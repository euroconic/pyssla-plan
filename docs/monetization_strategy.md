# Monetization Strategy 💰

How to sustain Pyssla Plan while keeping the core experience "Free & Accessible" for kids.

## 1. The "Passthrough" Model (Affiliates) 🛒
**Low Friction. High Relevance.**
*   **Concept:** The "Shopping List" feature we planned.
*   **Mechanism:** "You need 250 Red beads and 1 Pegboard." -> **[Buy on Amazon]**.
*   **Logic:** Users *cannot* use the output of your app without buying physical goods. You are the perfect referrer.
*   **Potential:** 3-5% commission on bead buckets/pegboards.

## 2. The "Power Parent" Tier (Freemium) ⭐️
**Keep the tool free. Charge for convenience/power.**
*   **Free:** Unlimited drawing, 1:1 printing (This drives growth).
*   **Pro ($10 One-Time or $2/mo):**
    *   **Unlimited AI:** "Magic Maker" costs you API fees. Limit free users to 3 generations/day. Pro gets unlimited.
    *   **Cloud Library:** Save unlimited designs to your account (Supabase cost). Free users get local storage only?
    *   **"Classroom Pack":** PDF export with "Student Name" fields for teachers.

## 3. The "Goodwill" Model (Donations) ☕️
**Leverage the "Indie Developer" story.**
*   **Mechanism:** "Buy Me a Coffee" / GitHub Sponsors.
*   **Message:** "I built this to keep it ad-free for my kids. Chip in to cover server costs."
*   **Fit:** Works extremely well with the Reddit/Open Source crowd ("Guerrilla Marketing").

## ❌ Avoid:
*   **Display Ads:** Google AdSense looks trashy and ruins the "Kid Safe" trust.
*   **Paywalling the Grid:** If they can't draw, they won't use it.

## Recommended Roadmap
1.  **Launch:** Totally Free + **Donation Button** (Builds trust/userbase).
2.  **Update 1:** Add **Affiliate Links** to the "Print/Export" screen (Passive income).
3.  **Update 2:** If AI costs get high, introduce **Pro Subscription** for heavy users.
